Cloudflare Tunnel Setup

A Cloudflare Tunnel exposes your local gateway to the internet. You can use a quick tunnel (temporary URL) or a named tunnel with a persistent domain.

Quick tunnel (no setup needed)

Leave the token field empty and click Start Tunnel. You will get a temporary *.trycloudflare.com URL.

Named tunnel (persistent domain)

  1. Install cloudflared: brew install cloudflared (macOS) or download here
  2. Log in: cloudflared tunnel login
  3. Create a tunnel: cloudflared tunnel create my-gateway
  4. Add a DNS route: cloudflared tunnel route dns my-gateway api.yourdomain.com
  5. Copy the tunnel token from the Cloudflare Zero Trust dashboard (Tunnels > your tunnel > Configure > Install connector)
  6. Paste the token below and click Save

AI Model Settings

The /v1/responses endpoint connects to aichat.uva.nl using your session cookie (auto-filled from login). Select a default model for requests that do not specify one.

UvA Extensions

Configure MCP extensions (Filesystem, Shell, Memory, etc.) so the UvA backend executes them server-side. When enabled, the Hermes prompt injection fallback is bypassed and UvA handles tool calls natively.

The UvA backend links extensions to chat threads. The thread_id must reference a thread that has your extensions (Filesystem, Shell, etc.) linked to it. The gateway reuses this thread ID so the backend loads those extensions for every request.
